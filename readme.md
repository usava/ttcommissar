## About this project

Предположим, что перед вами стоит задача, наблюдать за передвижениями ваших близких
людей. Скажем, детей. Вы хотите знать ходит ли Ваш сын в Москве в институт, а дочь в колледж в
Вашем родном Санкт-Петербурге. Прототип данной системы и предстоит разработать.
Внимание: данный прототип необходимо создать исключительно для проверки ваших знаний и
умений, а также понимания вашего видения организации инфраструктуры в целом. Данный
прототип никогда не будет использован в рабочих продуктах.
Ориентировочное время выполнения данной задачи – 4-5 часов.
Логическая составляющая:
Необходимо реализовать на LARAVEL справочник стран и городов, в справочнике должны быть
предусмотрены:
1. Справочник стран – хранит название страны
2. Справочник городов – хранит название городов, и координаты центра города.
Реализовать справочник «клиенты», должен содержать:
1. Ф.И.О.
2. Город – связь по ID с городом.
3. Email
4. Телефон
Реализовать справочник координат пользователя, должен представлять из себя таблицу со
списком:
1. ID клиента
2. Координаты пользователя
3. Дата обновления
В данном справочнике может храниться много координат к одному пользователю.
Интерфейс пользователя:
Интерфейс пользователя должен из себя представлять Яндекс или Google карту, на которой будет
видно маркеры пользователей (по последней их координате), при нажатии на маркер должно
открываться всплывающее окно с детальной информацией о клиента: ФИО, email и телефон для
связи.
На карте же, должен построиться маршрут пользователя по запомненным координатам, за
СЕГОДНЯШНИЙ ДЕНЬ.
Интерфейс необходимо сделать на Bootstrap 4, допустимо использовать любую из дефолтных тем
Bootstrap.
